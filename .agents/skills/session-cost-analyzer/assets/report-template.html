cost ≈ context size × steps

Every time the model takes a step — writing a reply, running a tool — it re-reads the whole conversation so far. So a session's price is driven less by what you asked than by how much was already in the conversation when you asked it: a one-line question at hour three can cost more than a big task at minute one. Each section below answers one question about this session; the note under each heading says how to read it.

Thinking — what the reasoning cost

Before most replies and tool calls, the model reasons privately. Those tokens are billed at the output rate — the priciest tier — but the text itself is never saved anywhere, so no tool can show it; interleaved is exactly that invisible share. The table shows which of your prompts set off the most reasoning. Many steps under one prompt usually means a debugging loop: every test run or log that comes back triggers a fresh round of thinking. The lever is fewer, bigger steps — batching commands, pushing retry loops into a subagent — not "think less".

Spending less next time

Three levers, in order of impact: 1. keep the main conversation small — /compact or start a fresh session between distinct phases; everything that stays in context is re-read on every later step (the "spent above 200k" card is what that would have saved here). 2. push heavy exploration and retry loops into subagents — they read in their own small context and report back. 3. batch independent commands into one step — every extra step re-reads the conversation and triggers a fresh round of billed reasoning.

Costs recomputed from raw tokens × LiteLLM prices (not Claude's reported cost). Token counts marked "est" are estimates (~4 characters per token); dollar totals come from exact billed token counts.
